# دليل التشغيل على جهاز/سيرفر داخلي 🖥️

الهدف: تشغيل اللوحة على جهاز داخل شبكة العمل، بحيث يفتحها مدير الإدارة وبقية المدراء
من أجهزتهم عبر رابط مثل `http://192.168.1.50:3000`.

> **مصطلح مهم:** "الجهاز المضيف" = الكمبيوتر أو السيرفر اللي بيشتغل عليه التطبيق وبيبقى مفتوحًا.

---

## الطريقة الأسهل: عبر Docker (موصى بها)

### 1) ثبّت Docker على الجهاز المضيف

- ويندوز / ماك: ثبّت **Docker Desktop** من https://www.docker.com/products/docker-desktop
- لينكس: ثبّت **Docker Engine** + **Docker Compose**

### 2) جهّز ملف الإعدادات

انسخ الملف وافتحه وعدّل إيميل مدير الإدارة:

```bash
cp .env.example .env
```

غيّر هذا السطر داخل `.env` إلى إيميل الدوام الخاص بمدير الإدارة:

```
ADMIN_EMAIL=boss@yourcompany.com
ADMIN_NAME=اسم مدير الإدارة
```

(إعدادات `SMTP_*` اختيارية — اتركها فارغة الآن لتجربة سريعة، الرمز بيظهر في سجلّ Docker.)

### 3) شغّل التطبيق بأمر واحد

```bash
docker compose up -d --build
```

خلاص! التطبيق يشتغل الآن ويعيد تشغيل نفسه تلقائيًا لو انطفأ الجهاز ورجع.

### 4) شارك الرابط

1. اعرف عنوان IP الخاص بالجهاز المضيف:
   - ويندوز: افتح `cmd` واكتب `ipconfig` وخذ الـ **IPv4 Address**.
   - ماك/لينكس: اكتب `hostname -I` أو `ifconfig`.
2. الرابط اللي ترسله للمدراء يكون: `http://<عنوان-IP>:3000`
   مثال: `http://192.168.1.50:3000`
3. لازم تكون أجهزتهم على **نفس شبكة العمل**.

### أوامر مفيدة

```bash
docker compose logs -f      # عرض السجل (ومنه رمز الدخول في وضع التجربة)
docker compose restart      # إعادة تشغيل
docker compose down         # إيقاف (البيانات تبقى محفوظة)
docker compose up -d --build   # إعادة التشغيل بعد تحديث الكود
```

> **البيانات محفوظة** في قرص Docker باسم `dashboard_data` ولا تُحذف عند إيقاف الحاوية.

## بديل: بدون Docker (تشغيل مباشر بـ Node.js)

لو الجهاز فيه **Node.js 20+** مثبّت:

```bash
npm install
cp .env.example .env       # عدّل ADMIN_EMAIL
npm run build
PORT=3000 HOSTNAME=0.0.0.0 npm start
```

ثم شارك الرابط `http://<عنوان-IP>:3000` كما في الأعلى.

> لإبقائه شغّالًا بشكل دائم استخدم أداة مثل **pm2**:
> `npm i -g pm2 && pm2 start "npm start" --name dashboard`

## ملاحظات مهمة

### فتح المنفذ في الجدار الناري (Firewall)
إذا ما قدر المدراء يفتحون الرابط، تأكد أن منفذ **3000** مسموح في جدار حماية الجهاز المضيف.

- ويندوز: أضف قاعدة وارد (Inbound Rule) للمنفذ 3000 في Windows Defender Firewall.
- لينكس (ufw): `sudo ufw allow 3000`

### تسجيل الدخول بالرمز (OTP)
- **وضع التجربة (بدون SMTP):** رمز الدخول يظهر في سجلّ التطبيق (`docker compose logs -f`).
  مناسب للتجربة، لكنه غير عملي للاستخدام الفعلي.
- **الاستخدام الفعلي:** عبّئ إعدادات `SMTP_*` في `.env` ببيانات بريد العمل، فيوصل
  الرمز إلى إيميل كل مستخدم تلقائيًا. (مثال Gmail موجود في `README.md`.)

### HTTPS (اختياري)
لو ربطت النظام لاحقًا بنطاق و HTTPS عبر بروكسي (Nginx / Caddy)، اجعل في `.env`:
```
COOKIE_SECURE=true
```

### أول دخول
افتح الرابط → أدخل `ADMIN_EMAIL` → خذ الرمز → ادخل كمدير إدارة → أضف المدراء من
تبويب **"المدراء والصلاحيات"**.
