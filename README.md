# داش بورد إدارة عمليات الأداء 📊

تطبيق ويب لقياس أداء القطاعات عبر مؤشرات استراتيجية (المستهدف مقابل المحقق لكل ربع)،
بواجهة رسومية، ونظام صلاحيات، وتسجيل دخول آمن **برقم الجوال عبر رمز (OTP)**.

> **نسخة العرض المباشرة:** https://perf-dashboard.onrender.com
> **للنشر داخليًا:** راجع [شرح-لتقنية-المعلومات.md](./شرح-لتقنية-المعلومات.md) و [DEPLOY.md](./DEPLOY.md)

## الهيكل

```
إدارة عمليات الأداء
└── القطاعات (حتى 7)
    └── لكل قطاع: 9 مؤشرات (قابلة للتعديل)
        └── لكل ربع: المستهدف + المحقق → نسبة الإنجاز (ملوّنة)
```

## الأدوار والصلاحيات

| الإجراء                          | مدير الإدارة | مدير القطاع |
| ------------------------------- | :---------: | :---------: |
| إدخال/تعديل قياسات قطاعاته        |     ✅      |     ✅      |
| رؤية كل القطاعات                  |     ✅      |     ❌      |
| إدارة القطاعات/المؤشرات/الفترات   |     ✅      |     ❌      |
| إدارة المدراء وإسناد القطاعات     |     ✅      |     ❌      |
| ضبط عتبات الألوان                 |     ✅      |     ❌      |

## المزايا

- **تسجيل الدخول برقم الجوال + OTP** — لا أحد يدخل إلا إذا أضافه مدير الإدارة.
- **النظرة العامة:** 5 بطاقات إحصائية + 9 عدّادات نصف دائرية (3×3) بالألوان.
- **الضغط على مؤشر** يعرض تفاصيله عبر القطاعات وكل الأرباع.
- **الضغط على بطاقة حالة** يبرز المؤشرات المطابقة ويُظلّل الباقي.
- **تصدير Excel** لكل البيانات.
- كل الهيكل والعتبات قابلة للتعديل من داخل اللوحة.

## التقنيات

- Next.js 15 (App Router) + TypeScript + React 19
- الرسوم والعدّادات بـ SVG مخصّص
- التخزين: **PostgreSQL** (عبر `DATABASE_URL`) أو ملف JSON محلي كاحتياط
- تسجيل الدخول: OTP برقم الجوال، وإرسال الرمز عبر بوابة SMS قابلة للتهيئة

## نسبة الإنجاز والألوان

نسبة الإنجاز = (المحقق ÷ المستهدف) × ١٠٠، وتُلوّن حسب عتبات قابلة للتعديل:

- 🟢 **وفق المسار:** ≥ 100% · 🟡 **متعثر جزئيًا:** 80–99% · 🔴 **متعثر:** < 80%

## التشغيل بضغطة واحدة (للتجربة على جهازك)

- **ويندوز:** انقر مرتين على `تشغيل-ويندوز.bat`
- **ماك:** انقر مرتين على `تشغيل-ماك.command`

## التشغيل عبر Docker (موصى به للنشر الداخلي)

```bash
cp .env.example .env          # عدّل ADMIN_PHONE و DATABASE_URL و SMS_*
docker compose up -d --build
```

ثم افتح `http://<عنوان-IP>:3000`.

## المتغيّرات الأساسية (.env)

| المتغير | الوصف |
| --- | --- |
| `ADMIN_PHONE` | جوال مدير الإدارة الأول |
| `DATABASE_URL` | رابط Postgres للحفظ الدائم (فارغ = ملف محلي) |
| `COOKIE_SECURE` | `true` خلف HTTPS |
| `SEED_DEMO` | `false` للاستخدام الفعلي |
| `SMS_PROVIDER` + بياناته | لإرسال الرمز فعليًا (`generic` أو `twilio`) |

راجع `.env.example` للقائمة الكاملة.

## البدء (لمدير الإدارة)

1. سجّل الدخول برقم `ADMIN_PHONE` (الرمز يصل بالرسالة، أو يظهر على الشاشة في وضع التجربة).
2. **الهيكل التنظيمي:** راجع القطاعات، أضف الجهات/الفترات، وعدّل المؤشرات والعتبات.
3. **المدراء والصلاحيات:** أضف مدراء القطاعات بأرقام جوالاتهم وأسند لكل واحد قطاعه.
4. يدخل كل مدير قطاع ويعبّئ القياسات، وتتجمّع عند مدير الإدارة في النظرة العامة.
