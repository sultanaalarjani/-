# لوحة تحكم مدير الإدارة 🗂️

لوحة تحكم ويب لإدارة فريق العمل بنظام صلاحيات:

- **مدير الإدارة (Admin):** صلاحيات كاملة — يضيف ويحذف المدراء، يحدد الحقول التي تُدخَل،
  ويشاهد **كل** البيانات التي يدخلها الجميع في مكان واحد (تتحدّث تلقائيًا).
- **المدراء (Managers):** صلاحية **إدخال البيانات فقط**، وكل مدير يشاهد إدخالاته هو فقط.
- **تسجيل الدخول بإيميل الدوام + رمز (OTP):** لا أحد يدخل إلا إذا كان إيميله مضافًا من
  مدير الإدارة، ويصله رمز مكوّن من 6 أرقام للتحقق.

## التقنيات

- Next.js 15 (App Router) + TypeScript + React 19
- تخزين البيانات في ملف JSON بسيط (بدون قاعدة بيانات خارجية)
- إرسال الرمز عبر البريد بـ Nodemailer (SMTP)
- تشغيل بضغطة واحدة عبر Docker

## 🚀 التشغيل على جهاز/سيرفر داخلي (ومشاركته مع المدراء)

لتشغيله على جهاز داخل شبكة العمل ومشاركة رابط مع مدير الإدارة، راجع الدليل الكامل خطوة بخطوة:
**[DEPLOY.md](./DEPLOY.md)**

باختصار عبر Docker:

```bash
cp .env.example .env          # عدّل ADMIN_EMAIL بإيميل مدير الإدارة
docker compose up -d --build  # شغّل التطبيق
```

ثم شارك الرابط `http://<عنوان-IP-للجهاز>:3000` مع المدراء على نفس الشبكة.

## التشغيل محليًا

```bash
# 1) ثبّت الحزم
npm install

# 2) جهّز ملف الإعدادات
cp .env.example .env
#   ثم افتح .env وعدّل:
#   - ADMIN_EMAIL : إيميل مدير الإدارة الأول (يُنشأ تلقائيًا)
#   - SESSION_SECRET : أي نص طويل عشوائي
#   - إعدادات SMTP لإرسال البريد (اختياري في التجربة)

# 3) للتطوير
npm run dev

# 4) للإنتاج
npm run build && npm start
```

ثم افتح: http://localhost:3000

## أول تسجيل دخول

1. افتح الموقع → أدخل `ADMIN_EMAIL` الذي وضعته في `.env`.
2. يصلك رمز الدخول:
   - **مع إعداد SMTP:** يصل إلى بريدك.
   - **بدون إعداد SMTP (وضع التجربة):** يظهر الرمز في شاشة الـ Terminal وفي صفحة الدخول.
3. بعد الدخول كمدير إدارة:
   - **المدراء والصلاحيات:** أضف إيميلات الدوام للمدراء (صلاحية إدخال فقط) أو مدراء إدارة آخرين.
   - **إعداد الحقول:** حدّد الحقول التي يعبّيها المدراء (نص / رقم / تاريخ / قائمة...).
   - **كل البيانات:** تابع كل ما يُدخله المدراء، مع اسم المُدخِل وتاريخ الإدخال.

## إعداد البريد (SMTP)

في ملف `.env` عبّئ بيانات بريد الدوام، مثال لـ Gmail:

```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=youraddress@gmail.com
SMTP_PASS=app-password-here
SMTP_FROM=youraddress@gmail.com
```

> لو تركت إعدادات SMTP فارغة، يعمل النظام في **وضع التجربة** ويعرض رمز الدخول في الـ Terminal
> بدل إرساله بالبريد — مناسب للتجربة قبل ربط البريد الحقيقي.
