# داش بورد إدارة عمليات الأداء 📊

لوحة تحكم لقياس أداء الجهات عبر مؤشرات، بنظام صلاحيات وتسجيل دخول آمن.

## الهيكل التنظيمي

```
إدارة عمليات الأداء
└── القطاعات (حتى 7 قطاعات)
    ├── قطاع البنية التحتية
    ├── قطاع الخدمات الاجتماعية
    ├── قطاع المالي والاقتصادي
    └── قطاع الشؤون الحكومية
        └── الجهات (تابعة لكل قطاع)
            └── المؤشرات (9 افتراضية، قابلة للإضافة/الحذف/التعديل)
                └── قياس لكل فترة: المستهدف + المحقق → نسبة الإنجاز
```

## الأدوار والصلاحيات

| الإجراء                         | مدير الإدارة | مدير القطاع |
| ------------------------------- | :---------: | :---------: |
| إدخال/تعديل قياسات قطاعاته       |     ✅      |     ✅      |
| رؤية بيانات قطاعاته فقط          |      —      |     ✅      |
| رؤية كل القطاعات والجهات         |     ✅      |     ❌      |
| إدارة القطاعات والجهات والمؤشرات |     ✅      |     ❌      |
| إدارة الفترات الزمنية            |     ✅      |     ❌      |
| إضافة المدراء وإسناد القطاعات    |     ✅      |     ❌      |

- **مدير الإدارة:** صلاحيات كاملة، يبني الهيكل (قطاعات/جهات/مؤشرات/فترات)، يضيف المدراء
  ويُسند لكل مدير قطاعاته، ويشاهد **النظرة العامة** لكل الأداء مع التلوين والتصدير.
- **مدير القطاع:** يدخل ويعدّل قياسات **جهات قطاعاته فقط**، ولا يرى بيانات القطاعات الأخرى.
- **تسجيل الدخول:** بإيميل الدوام + رمز (OTP). لا أحد يدخل إلا إذا أضافه مدير الإدارة.

## المؤشرات التسعة الافتراضية

1. نسبة الأجهزة العامة التي يتم قياس خدماتها (%)
2. نسبة الأجهزة ذات الأداء المنخفض التي تم عقد جلسات مراجعة لها (%)
3. نسبة التقارير الممتثلة لمعايير جودة ملاحظات الأداء (%)
4. نسبة قابلية قياس مؤشرات المخرجات الوطنية (%)
5. نسبة قابلية قياس الاستراتيجيات الوطنية المعتمدة من مجلس الوزراء (%)
6. عدد الأجهزة العامة التي تم قياس استراتيجياتها المؤسسية (عدد)
7. نسبة التكليفات المباشرة المكتملة أو على المسار (%)
8. نسبة وجودة وفعالية معالجة طلبات التغيير وتحديث سير التقدم للمؤشرات والمبادرات (%)
9. نسبة إتمام الخطة الفردية للاحتياج التطويري (%)

> جميع المؤشرات قابلة للإضافة/الحذف/التعديل من تبويب **الهيكل التنظيمي ← المؤشرات**.

## نسبة الإنجاز والتلوين

لكل قياس يُدخَل **المستهدف** و**المحقق**، وتُحسب نسبة الإنجاز = (المحقق ÷ المستهدف) × ١٠٠،
وتُلوّن تلقائيًا:

- 🟢 **ممتاز:** الإنجاز ≥ 100%
- 🟡 **جيد:** 80% – 99%
- 🔴 **متعثر:** أقل من 80%

## التقنيات

- Next.js 15 (App Router) + TypeScript + React 19
- تخزين البيانات في ملف JSON بسيط (بدون قاعدة بيانات خارجية)
- إرسال رمز الدخول بـ Nodemailer (SMTP)
- تشغيل بضغطة واحدة عبر Docker

## 🚀 التشغيل على جهاز/سيرفر داخلي (ومشاركته مع المدراء)

الدليل الكامل خطوة بخطوة في **[DEPLOY.md](./DEPLOY.md)**. باختصار عبر Docker:

```bash
cp .env.example .env          # عدّل ADMIN_EMAIL بإيميل مدير الإدارة
docker compose up -d --build  # شغّل التطبيق
```

ثم شارك الرابط `http://<عنوان-IP-للجهاز>:3000` مع المدراء على نفس الشبكة.

## التشغيل محليًا

```bash
npm install
cp .env.example .env       # عدّل ADMIN_EMAIL و SESSION_SECRET
npm run dev                # للتطوير
# أو للإنتاج:
npm run build && npm start
```

ثم افتح: http://localhost:3000

## خطوات البدء (لمدير الإدارة)

1. ادخل بإيميل `ADMIN_EMAIL` (الرمز يصل بالبريد، أو يظهر في الـ Terminal بوضع التجربة).
2. **الهيكل التنظيمي:** تأكد من القطاعات، أضف **الجهات** لكل قطاع، راجع **المؤشرات**،
   وأضف **الفترات** الزمنية.
3. **المدراء والصلاحيات:** أضف إيميل كل مدير قطاع وأسند له قطاعه/قطاعاته.
4. يدخل كل مدير قطاع ويعبّئ القياسات (المستهدف/المحقق) لجهاته.
5. تابع **النظرة العامة** لكل الأداء، وصدّر النتائج إلى Excel.

## إعداد البريد (SMTP)

في `.env` عبّئ بيانات بريد الدوام، مثال Gmail:

```
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=youraddress@gmail.com
SMTP_PASS=app-password-here
SMTP_FROM=youraddress@gmail.com
```

> لو تركت إعدادات SMTP فارغة، يعمل النظام في **وضع التجربة** ويعرض رمز الدخول في الـ Terminal.
