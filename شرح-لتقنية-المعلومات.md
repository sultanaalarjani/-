# لوحة "إدارة عمليات الأداء" — دليل تقنية المعلومات (للنشر)

## ما هو التطبيق؟
لوحة تحكم ويب لقياس أداء القطاعات والجهات عبر مؤشرات استراتيجية (المستهدف مقابل المحقق
لكل ربع)، مع صلاحيات للمستخدمين وتسجيل دخول آمن بإيميل الدوام عبر رمز (OTP).

- **مدير الإدارة (Admin):** يرى كل القطاعات والمؤشرات، يدير الهيكل والمستخدمين.
- **مدير القطاع (Manager):** يدخل بيانات قطاعاته فقط.

## التقنيات
- **الإطار:** Next.js 15 (React 19) + TypeScript.
- **التخزين:** ملف JSON محلي (`data/db.json`) — لا حاجة لقاعدة بيانات خارجية.
- **البريد:** Nodemailer لإرسال رمز الدخول عبر SMTP.
- لا يعتمد على أي خدمة سحابية خارجية؛ يعمل بالكامل داخل الشبكة.

## المتطلبات
- خيار (أ): **Docker + Docker Compose** (موصى به). أو
- خيار (ب): **Node.js 20+**.
- منفذ TCP **3000** متاح (قابل للتغيير).
- مساحة قرص بسيطة جدًا.

## النشر — الخيار (أ): Docker (الأسهل)
```bash
# داخل مجلد المشروع
cp .env.example .env        # ثم عدّل المتغيرات (انظر أدناه)
docker compose up -d --build
```
- يعمل تلقائيًا ويعيد التشغيل بعد إعادة إقلاع الجهاز (`restart: unless-stopped`).
- البيانات تُحفظ في فوليوم باسم `dashboard_data` (دائمة).

## النشر — الخيار (ب): بدون Docker
```bash
npm install
npm run build
npm start            # يستمع على http://0.0.0.0:3000
```
لإبقائه دائمًا: `npm i -g pm2 && pm2 start "npm start" --name perf-dashboard && pm2 save`.

## متغيّرات البيئة (ملف .env)
| المتغير | الوصف |
| --- | --- |
| `ADMIN_EMAIL` | إيميل مدير الإدارة الأول (يُنشأ تلقائيًا عند أول تشغيل) |
| `ADMIN_NAME` | اسم مدير الإدارة (اختياري) |
| `COOKIE_SECURE` | `false` للتشغيل عبر HTTP الداخلي، `true` خلف HTTPS |
| `SMTP_HOST` / `SMTP_PORT` | خادم البريد ومنفذه (مثال 587) |
| `SMTP_USER` / `SMTP_PASS` | بيانات اعتماد بريد الإرسال |
| `SMTP_FROM` | عنوان المُرسِل الظاهر |

> بدون إعداد SMTP يعمل النظام في "وضع التجربة" ويعرض الرمز في السجل بدل إرساله بالبريد.

## مشاركة الرابط مع المستخدمين
بعد التشغيل، الرابط يكون `http://<عنوان-IP-للسيرفر>:3000`. يجب فتح المنفذ 3000 في
الجدار الناري للسيرفر، والمستخدمون على نفس الشبكة (أو عبر VPN العمل).

## الأمان والنسخ الاحتياطي
- الدخول مقصور على الإيميلات التي يضيفها مدير الإدارة فقط (OTP).
- **النسخ الاحتياطي:** خذ نسخة دورية من `data/db.json` (أو فوليوم `dashboard_data`).
- للإنتاج عبر نطاق: ضع التطبيق خلف **Nginx/Caddy** مع HTTPS واضبط `COOKIE_SECURE=true`.

## ملاحظات وتوسيع مستقبلي

  الخوادم؛ البنية الحالية كافية للاستخدام ضمن إدارة واحدة.
